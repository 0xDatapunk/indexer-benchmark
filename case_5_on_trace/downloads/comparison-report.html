9. Subsquid Data Fix

Issue: The original Subsquid implementation had problems with the to address and path fields being set to zero addresses instead of properly extracting them from the transaction call data.

Fix Applied: The Subsquid indexer implementation was updated to properly decode the transaction calldata. Specifically:

  • The to address is now extracted from bytes 192-256 of the calldata (taking the last 20 bytes)
  • The path array is extracted by calculating the offset in the calldata based on the path pointer, then reading each address
  • The deadline parameter is now properly read from the calldata

These changes ensure that Subsquid correctly captures all the necessary fields for Uniswap V2 swaps, with proper addresses for to and token path entries.
